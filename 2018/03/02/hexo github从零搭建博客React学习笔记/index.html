    新建git仓库

新建项目，项目必须要遵守格式：

账户名.github.io，

不然接下来会有很多麻烦。( 本文这里已经注册过了所以这里只是演示)

在建好的项目右侧有个settings按钮，点击它，

向下拉到GitHub Pages，

你会看到那边有个网址，就是你的博客地址

在自己认为合适的地方创个文件夹，我这里新建blog文件夹,通过终端进入该文件，通过以下命令安装hexo

检查hexo是否安装成功

看到Start blogging with Hexo就说明成功了

  • 进阶

主题的配置：NexT

  1. 安装NexT主题
  1. 在站点目录中(blog)，打开配置文件_config.yml，找到theme，修改
  1. 启动服务，验证主题

如果看到如下界面，说明成功安装了NexT主题

4.主题相关的设定

  • 主题布局

在项目文件 themes/next 主题目录下的_config.yml文件中

将scheme设定为Pisces

  • 语言设定

在站点根目录下修改配置文件_cofig.yml中的language为zh-Hans（简体中文）

  • 修改菜单项-增加菜单项

    在 themes/next 主题目录下修改配置文件_cofig.yml中的menu，

    增添一个菜单名称 这里以example为例

example:/example

（若你的站点运行在子目录中，请将链接前缀的 / 去掉）

这些配置都要与你主题目录下的languages文件中对应的yml文档里配置相关联。

比如你在站点根目录中的配置文件设置language为zh-Hans，那么就要进入到主题目录下的languages文件中修改zh-Hans.yml，这样才能显示出菜单项新增的中文内容

  • 设置菜单项图标

    对应的字段是menu_icons。

注意：

以前的版本中，设置图标
格式为item name：icon name

现在可直接在链接后添加修改

icon name是Font Awesome图标的名字

而 enable 可用于控制是否显示图标，你可以设置成 false 来去掉图标。

  • 设置侧栏位置

修改主题目录下sidebar的position值

  • 设置头像

在站点根目录下载配置文件中新增avatar

地址可以是网络地址

也可以是本地地址（放置在source/images/ 目录下）

  • 设置文章代码主题

在主题目录下修改配置文件 highlight_theme，默认值为nomal。可以设置为night

  • 拓展

    添加留言板
    其实现在的主题里都兼容了各种主流留言板，如（畅言、gitment、valine、disqus）无需过多操作

本文用的valine

按照如上配置将enable改为true，gitment也是如此，appid与key值去相关网站注册获取，再按照官方文档修改想要的修改的配置
